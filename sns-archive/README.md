# 統合SNSアーカイブ

Twitter（X）の公式アーカイブ、Misskey、Bluesky、Mastodon の自分の投稿を
**1つのタイムラインに統合**して、時系列で閲覧・全文検索できるローカルアプリです。
Twitterアーカイブ付属のHTMLビューアの上位互換を目指しています。

- **「投稿・RT」と「いいね・お気に入り」の2タブ**で切り替え表示
- **各サービスとも複数アカウント対応**（Twitter 3垢 + Misskey 4垢…のような構成もOK）
- コマンド付き全文検索（`since:` `has:image` `-除外` `OR` など）
- **ライトモード / ダークモード両対応**（OS設定に自動追従、左上のボタンで手動切替）
- 月別ナビゲーション（サービス別の積み上げバーつき）
- 画像・動画は可能な限りローカル保存(オフラインでも閲覧可)
- 「この日の過去」「ランダム表示」などの思い出機能
- エクスポートファイル取り込みでも、欠けた本文・画像を**公開APIから自動補完**
- 投稿に**ブックマーク（自由な名前のラベル）**を付けて整理・絞り込み
- データはすべて手元の `archive.db` と `media/` フォルダに保存。取得先は各SNSのみ

---

## 1. 必要なもの

- Windows 11（日本語版。batファイルはShift_JISで保存されています）
- Python 3.10 以上 … https://www.python.org/downloads/ から入手し、
  インストーラで **「Add python.exe to PATH」に必ずチェック** を入れてください
- 各サービスの認証情報（下記 3章）

## 2. セットアップ

1. このフォルダを好きな場所に置く（例: `C:\sns-archive`）
2. `setup.bat` をダブルクリック（仮想環境の作成とライブラリ導入を自動で行います）
3. できあがった `config.json` をメモ帳等で編集

> `config.json` にはトークン類を平文で保存します。フォルダごと他人に渡さないでください。

### 複数アカウントの書き方

すべてのサービスが `[ ]`（リスト）形式です。**`{ }` ブロックをカンマで区切って並べるだけ**で
何アカウントでも増やせます。使わないサービスは `[]` に、使わない例示ブロックは削除してください。

```json
"twitter": [
  { "archive_dir": "C:/Users/you/Downloads/twitter-main.zip",  "account_label": "" },
  { "archive_dir": "C:/Users/you/Downloads/twitter-sub.zip",   "account_label": "@サブ垢" }
],
"bluesky": [
  { "handle": "main.bsky.social", "app_password": "xxxx-xxxx-xxxx-xxxx" },
  { "handle": "sub.bsky.social",  "app_password": "yyyy-yyyy-yyyy-yyyy" }
]
```

`label` / `account_label` は表示名（空なら自動で `@ユーザー名@ホスト` などになります）。
取り込んだアカウントはビューアの左サイドバーに並び、個別にオン/オフできます。

## 3. 各サービスの準備

### Twitter（X）

1. X の 設定とプライバシー → アカウント → **データのアーカイブをダウンロード** を申請
   （用意されるまで1日程度かかります）
2. 届いた **zipファイルのパス** を `config.json` の `twitter` に書く
   （zipのままでOK。初回に自動で展開します。展開済みフォルダの指定も可）

パスの区切りは `/` か `\\` を使ってください（`\` 1つだとエラーになります）。
複数アカウント分のzipがある場合は、上の例のようにブロックを並べます。

### Misskey（APIで全自動取得・推奨）

手動エクスポートではなく **APIでの自動取得** を採用しています。理由:

- エクスポートJSONには **リノート先の本文・リアクション（いいね）履歴が含まれない**
- APIなら再実行するだけで **差分同期** ができる（新しい投稿だけ追加取得）
- メディアの取得漏れが起きにくい

各サーバで以下を行ってください:

1. Misskeyの **設定 → API → アクセストークンの発行**
2. 権限は自分専用なので「**すべて**」にチェックで構いません
   （最小にしたい場合は「アカウントの情報を見る」「リアクションを見る」あたりの読み取り系）
3. 発行されたトークンを `config.json` の該当サーバの `token` に貼る

いいね（リアクション）履歴が取得できない場合は、そのサーバの
**設定 → プライバシー → リアクション履歴を公開する** を有効にしてから再実行してください。

**エクスポートファイルから取り込む場合**（サーバが閉鎖済みなどでAPIが使えないとき）:
設定 → その他 → エクスポート で作った「ノート」のファイル（.json または .zip）を指定します。

```json
"misskey": [
  { "archive": "C:/Users/you/Downloads/notes-xxxx.zip",
    "host": "misskey.example.jp", "user": "あなたのユーザーID" }
]
```

エクスポートに含まれないリノート先の本文・画像は、**サーバの公開APIから自動で補完**します
（トークン不要。取得できなかった分は再実行で続きから補完）。
ただし**リアクション（いいね）履歴はエクスポート自体に含まれない**ため、APIモードでのみ取得できます。

**token と archive の併用（おすすめ）**: 両方を書いておくと、**初回だけアーカイブから
一括で取り込み、それ以降の新しい分だけをAPIで取得**します。投稿が数万件あるアカウントでも、
初回のAPI待ち時間を大幅に短縮できます（リアクション履歴は初回もAPIで取得します）。

```json
"misskey": [
  { "host": "misskey.example.jp", "token": "アクセストークン",
    "archive": "C:/Users/you/Downloads/notes-xxxx.zip" }
]
```

### Bluesky

1. Blueskyアプリの **設定 → プライバシーとセキュリティ → アプリパスワード** で新規発行
   （本パスワードではなく、必ずアプリパスワードを使ってください）
2. `config.json` の `bluesky` に handle と app_password を記入

**エクスポートファイルから取り込む場合**:
設定 → アカウント → **アカウントデータのエクスポート** でダウンロードできる
リポジトリファイル（`.car`、またはそれを含むzip）を指定します。

```json
"bluesky": [
  { "archive": "C:/Users/you/Downloads/repo.car" }
]
```

.car には自分の投稿（本文・日時・いいねやリポストの正確な日時を含む）が入っており、
本文が入っていない「いいね・リポストの元投稿」やメディアは
**Blueskyの公開API・公開エンドポイントから自動で補完**します（アプリパスワード不要）。
補完まで含めるとAPI取得とほぼ同等の内容になります。

### Mastodon

**A) API取得（推奨）** — 投稿・ブースト・お気に入りをすべて本文つきで取得でき、差分同期も可能です。

1. Mastodonの **設定 → 開発 → 新規アプリ**（アプリ名は何でもOK、権限は `read` だけで十分）
2. 作成したアプリを開き、**「アクセストークン」** をコピー
3. `config.json` に記入:

```json
"mastodon": [
  { "host": "mstdn.example.jp", "token": "アクセストークン", "label": "" }
]
```

**B) アーカイブ（エクスポートzip）取り込み** — 設定 → データのエクスポート →
「投稿とメディアのアーカイブを請求」で作られるzipを取り込めます:

```json
"mastodon": [
  { "archive": "C:/Users/you/Downloads/archive-20260101-xxxx.zip" }
]
```

アーカイブにはブースト・お気に入りの本文が含まれませんが、**元投稿があるサーバの
公開APIから自動で補完**します（お気に入りの日時は元投稿IDからの推定値。「ごろ」表示）。
元投稿が削除済み・非公開の場合は補完できないため、可能ならAPI取得をおすすめします。

**token と archive の併用（おすすめ）**: 両方を書いておくと、**初回だけアーカイブから
一括で取り込み、それ以降の新しい分だけをAPIで取得**します。エクスポートとAPIでブースト・
お気に入りのID体系が異なる問題も自動で突き合わせて統合するので、重複は生じません。

## 4. 取り込み

`sync_all.bat` をダブルクリック（Twitter → Misskey → Bluesky → Mastodon の順に実行）。
個別にやり直したいときは `sync_twitter.bat` / `sync_misskey.bat` /
`sync_bluesky.bat` / `sync_mastodon.bat`。

- 投稿やメディアが多いと初回は時間がかかります（レート制限対策で少し待ちながら取得します）
- **途中で中断しても大丈夫**。再実行すれば続きから取り込まれます
- **2回目以降は差分だけ取得**されるので、定期的に回せば常に最新のアーカイブになります
- 過去の同期で一部を取りこぼしていた場合でも、次回以降の同期で自動的に穴埋めされます
  （最新から前回取得済みの範囲まで必ず遡って照合するため）
- Twitterは新しいアーカイブzipのパスに書き換えて再実行すれば、新規分だけが増えます
  （過去に取り込んだ分は残ります）

### 差分同期のしくみ（2回目以降が速い理由）

各サービスとも、**初回だけ全期間を取得**し、完了すると「全件取得済み」の印を付けます。
2回目以降は新しい側から確認し、**取り込み済みの投稿が200件連続した時点で打ち切る**
差分取得になるため、数万件のアカウントでも数回のAPIアクセスで終わります
（Misskeyは加えて、前回取得済みの最新時期まで確認した時点でも終了します）。

- 同期が途中で失敗した場合は自動的に印が外れ、**次回は全期間を確認して穴を自己修復**します
- 過去の日時を持つ投稿があとから増えるケース（他サーバからのインポート等）は
  差分では拾えません。その場合は全件を取り直してください:

```bat
sync_misskey.bat --full        （bluesky / mastodon も同様）
```

Twitterはアーカイブファイルからの取り込みのため、毎回ファイル全体を読み直します
（差分の概念はありません。既存分は上書きされるだけなので安全です）。

## 5. 閲覧

`start.bat` をダブルクリック → ブラウザで `http://127.0.0.1:5089/` が開きます。
終了は起動した黒いウィンドウで `Ctrl+C`（または閉じる）。

- 上部の **「投稿・RT」/「いいね・お気に入り」タブ**で表示を切り替え
- 左上の **☀/☾ ボタン**でライト/ダークを切り替え（初期値はOSの設定に追従）
- 左サイドバー: アカウント別・種類別のフィルタ、並び順、月別ナビ
- 検索欄の下に、表示中の件数と**アカウント別の内訳**（サービスの色付き）を表示
- 検索ボックスは `/` キーでフォーカス、入力すると自動で絞り込み（Enterで即実行）
- 画像クリックで拡大表示、右下の「↑」ボタンでいちばん上へ戻る
- 「この日の過去」= 過去全年の今日と同じ日付の投稿を表示

### ブックマーク（ラベル）

各投稿のフッターにある **🔖 ボタン**から、「お気に入り」「大事な話」「レシピ用」など
**自由な名前のラベル**をいくつでも付けられます（1つの投稿に複数可）。

- 付けたラベルは投稿カードの上部にタグとして表示され、**タグをクリックするとそのラベルで絞り込み**
- 左サイドバーの「ブックマーク」に全ラベルが件数つきで並び、クリックで絞り込み。
  「🔖 すべてのブックマーク」で何かしらラベルを付けた投稿だけを表示
- 検索とも併用可能（例: ラベル「大事な話」で絞り込み中に `service:twitter 仕事`）
- ブックマークは `archive.db` に保存されます（後述のバックアップ対象）

### 前後の投稿（文脈表示）

各投稿のフッターにある **「⏲ 前後」ボタン**を押すと、その投稿を中心に
**同じアカウントの時系列**が前後10件ずつ表示されます。上端の「▲」でさらに新しい側、
下端の「▼」でさらに古い側を10件ずつ読み足せるので、「この投稿をした頃、
自分は何を書いていたか」をどこまでも辿れます。文脈の中の別の投稿から
さらにその前後を開き直すこともできます（画像の拡大やブックマークもその場で操作可）。

## 6. 検索コマンド一覧

スペース区切りはAND。ビューアの「?」ボタンからも参照できます。

| コマンド | 意味 |
|---|---|
| `ねこ 写真` | 両方の語を含む（AND） |
| `ねこ OR いぬ` | どちらかを含む。`ねこ OR いぬ かわいい` = (ねこ∨いぬ)∧かわいい |
| `"おはようございます"` | フレーズ検索 |
| `-ラーメン` | この語を含む投稿を除外 |
| `service:twitter` | サービス指定（`twitter` `misskey` `bluesky` `mastodon`、別名 `x` `mi` `bsky` `mstdn` `md`）。`-service:` で除外。2つ並べるとOR |
| `account:misskey.io` | アカウント名の部分一致 |
| `from:username` | RT・いいねの元投稿者名で絞る（表示名でも可） |
| `since:2020-01-01` | この日以降。`since:2020` `since:2020-04` も可 |
| `until:2021-06-30` | この日まで |
| `date:2019-05-01` | その日のみ。`date:05-01` で全年の5月1日 |
| `has:image` | 画像つき。`has:media`（何かしら添付） `has:video` `has:link` |
| `type:like` | 種類指定（`post` `repost` `like`）。`is:rt` `is:reply` `-is:reply` なども可 |
| `label:大事な話` | 付けたブックマーク（ラベル）で絞る。`label:*` で何かラベルが付いたもの全部、`-label:名前` で除外 |
| `has:bookmark` | 何かしらブックマークした投稿（`label:*` と同じ） |
| `sort:asc` | 古い順。`sort:random` でランダム |

検索は投稿本文に加え、**画像の説明文（alt/キャプション）** も対象です。
タブと検索は同時に効きます（例: いいねタブで `service:mastodon 猫`）。

### 検索の高速化（自動）

日本語の部分一致検索を高速化するため、**全文検索インデックス（SQLite FTS5 trigram）を
自動で作成・維持**します。設定は不要です。

- 既存の `archive.db` でも、次回起動・同期時に自動で構築されます
  （15万件で1秒弱。「検索インデックスを構築しています…」と表示されます）
- 3文字以上の検索語で効きます（15万件で約0.4秒 → 約0.1秒。ヒットなしは一瞬）。
  1〜2文字の語は仕組み上インデックスが効かず従来どおりですが、実用上問題ない速度です
- インデックスの分だけ `archive.db` は少し大きくなります（本文サイズの2倍前後）
- 万一 FTS5 が使えない環境（古いPython同梱のSQLite等）では、**自動で従来のLIKE検索に
  切り替わり**、機能はそのまま使えます。検索結果はどちらの方式でも同一です

## 7. フォルダ構成とバックアップ

```
sns-archive/
├─ config.json        … 設定（トークン含む・取扱注意）
├─ archive.db         … 全投稿データ（SQLite）
├─ media/             … ダウンロードしたメディア
│   ├─ twitter/  misskey_<ホスト名>/  bluesky/  mastodon_<ホスト名>/
├─ ingest_*.py  app.py  common.py
└─ *.bat
```

**`archive.db` と `media/` の2つをコピーすればアーカイブ全体のバックアップになります。**

zip や .car を指定して取り込んだ場合、`*_archive_extracted/` に一時展開されますが、
**取り込みが完了すると自動で削除**されます（メディアは先に `media/` へ保存済みなので失われません）。
展開データを残したい場合は、config.json に `"keep_extracted": true` を足すか、各 sync に
`--keep-extracted` を付けてください。
ゼロから取り込み直したい場合は `archive.db` を削除して sync を再実行
（メディアは残るので再ダウンロードは最小限で済みます）。
**ただし `archive.db` には手作業で付けたブックマークも入っています。**削除すると
ブックマークも消えるので、消す前にバックアップを取っておくことをおすすめします。

## 8. 仕様上の制限（各サービス側の事情によるもの）

- **Twitterのいいねの日時**: アーカイブに「いいねした日時」が含まれないため、
  元ツイートIDから **投稿された日時** を復元して配置しています（表示に「ごろ」と付きます）。
  ごく古いツイート（2010年以前）は日時不明扱いになり、月別ナビの「日付なし」に入ります
- **TwitterのRT**: アーカイブ仕様により本文が140字前後で切り詰められており、
  元投稿の画像も含まれません
- **Twitterのいいねのメディア・投稿者名**: アーカイブに含まれないため取得できません
- **Mastodonのお気に入りの日時**: API側に「お気に入りした日時」が無いため、
  元投稿の投稿日時で配置しています（「ごろ」表示）
- **Mastodonのアーカイブ取り込み**: ブースト・お気に入りの本文が含まれないため、
  元サーバの公開APIから自動補完します。元投稿が削除済み・非公開・鍵付きの場合は補完できません
- **Misskeyのアーカイブ取り込み**: リアクション（いいね）履歴はエクスポートに含まれず、
  補完もできません（APIモードなら取得可能）
- **Blueskyで他人の動画**: ストリーミング形式のためサムネイルのみ保存し、
  再生は元投稿へのリンクになります（自分の投稿の動画はファイル本体を保存します）
- 元投稿が削除済みの場合、いいね・リポストは「削除済み」表示になります
- アイコン画像・フォロワー情報などは対象外です（投稿アーカイブに特化）

## 9. トラブルシューティング

**Q. 同期のたびに全件を取得しているようで時間がかかる**
初回（または途中失敗の直後）は全期間を確認するため時間がかかりますが、
正常に完了すれば次回から差分取得になります。毎回長い場合は、同期ログの
末尾にエラーが出ていないか確認してください（失敗があると安全のため
次回も全期間を確認します）。


- **`python` が見つからない** … Pythonインストール時にPATHチェックを忘れた可能性。
  再インストールするか、`py -3` が使えるならそのままsetup.batが拾います
- **batの文字が化ける** … batはShift_JIS（日本語版Windows標準）で保存しています。
  英語版Windows等で使う場合は `chcp 932` を実行してから使うか、Windows Terminalをお試しください
- **Misskey/Mastodonで429エラーが続く** … レート制限です。自動で待機・再開しますが、
  頻発する場合は時間をおいて再実行してください（差分から再開されます）
- **Mastodonで認証エラー** … トークンが正しいか、アプリの権限に `read` があるか確認
- **Blueskyでログイン失敗** … 本パスワードではなく「アプリパスワード」か確認。
  ハンドルは `@` なしでも可
- **ポート5089が使用中** … `config.json` の `port` を変更（例: 5090）して start.bat 再実行
- **ビューアに何も出ない** … タブ（投稿/いいね）と左の「種類」フィルタ、検索ボックスを確認。
  取り込み済み件数はサイドバー上部に表示されます
- **十数万件でも快適に動きます** … 月別ナビや絞り込みは高速化済みです。
  本文の全文検索だけは件数に比例して少し時間がかかりますが（数十万件で1秒未満）、
  サービスや期間で先に絞ると速くなります
- **古いDBもそのまま使えます** … 旧バージョンで作った `archive.db` は、初回起動時に
  自動で新形式へ更新されます（月別集計を高速化する索引の追加）。手動作業は不要です
